## 定义

环境管理（Environment Management）是指为长运行智能体设置和维护一个结构化的开发环境，包括功能列表、测试框架和进度跟踪，以支持智能体在多个会话中持续工作。

## 核心组件

### 1. 功能列表（Feature list）

**目的**：解决智能体"一次性完成"或过早宣布完成的问题

**实现**：
- 初始化智能体创建功能需求文件，扩展用户初始提示
- 所有功能初始标记为"failing"
- 使用JSON格式（比Markdown更不易被模型误改）
- 智能体只能通过更改 `passes` 字段来更新状态

**示例**：
```json
{
  "category": "functional",
  "description": "New chat button creates a fresh conversation",
  "steps": [
    "Navigate to main interface",
    "Click the 'New Chat' button",
    "Verify a new conversation is created",
    "Check that chat area shows welcome state",
    "Verify conversation appears in sidebar"
  ],
  "passes": false
}
```

**指令**：使用强措辞如"不可接受删除或编辑测试，因为这可能导致缺失或有bug的功能"

### 2. 进度跟踪

**组件**：
- `claude-progress.txt` 文件记录进度日志
- Git提交历史
- 结构化的进度更新

**实践**：
- 智能体在代码更改后将环境保持在干净状态
- 使用git提交和进度文件记录
- 允许智能体使用git回滚不良更改

### 3. 测试框架

**目的**：解决智能体过早标记功能为完成的问题

**实现**：
- 使用浏览器自动化工具（如Puppeteer MCP）进行端到端测试
- 截图功能帮助识别非代码层面的bug
- 智能体必须自我验证所有功能，仅在仔细测试后标记为"passing"

**示例**：
- 初始化智能体编写 `init.sh` 脚本运行开发服务器
- 编码智能体运行基本端到端测试
- 使用Puppeteer MCP启动新聊天、发送消息、接收响应

## 环境设置流程

### 初始化智能体

1. 创建功能需求文件（JSON格式）
2. 创建 `init.sh` 脚本运行开发服务器
3. 创建 `claude-progress.txt` 文件记录进度日志
4. 创建初始git提交

### 编码智能体

1. 读取 `init.sh` 脚本
2. 读取git日志和进度文件了解最近工作
3. 读取功能列表文件，选择最高优先级的未完成功能
4. 运行基本测试确保环境正常
5. 实现单个功能
6. 更新功能列表状态
7. 提交git和更新进度文件

## 干净状态原则

- 代码更改后将环境保持在干净状态
- 没有主要bug，代码有序且有良好文档
- 开发人员可以轻松开始新功能，无需先清理不相关的混乱
